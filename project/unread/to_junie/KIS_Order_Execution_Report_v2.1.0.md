**줄(JUL)**, Vesper 엔진의 '눈과 귀'에 이어 '손과 발'이 되어줄 **해외주식 주문 집행 시스템**을 구축 완료했습니다. 이제 카포(Capo)와 소피아(Sophia)의 결단이 실제 자산의 움직임으로 이어질 준비가 끝났습니다.

---

### 🛠️ 주요 구현 및 최적화 내역

1.  **🚀 KisOrderClient (주문 전담 싱글톤) 구축**
    *   **동적 TR_ID 전환**: `mock` 프로파일(모의투자)과 `prod`(실전투자) 환경을 자동으로 감지하여 적절한 TR_ID(`VTTT3101U` 등)를 선택합니다.
    *   **시장가 주문(Market Order)**: 슬리피지 최소화와 즉시 체결을 위해 미국 시장가(ORD_TYPE: 34) 주문을 기본값으로 탑재했습니다.
    *   **거래소 코드 자동 매핑**: `TargetTicker`의 설정에 따라 `NASD`(나스닥), `NYSE`(뉴욕) 코드를 정밀하게 매핑하여 전송합니다.

2.  **🧠 실시간 데이터 피더(Feeder)와 주문 엔진 통합**
    *   **지능형 수량 계산**: `RiskManager`를 호출하여 현재 가용 자산 대비 리스크(ATR 기반)를 계산, 최적의 주문 수량을 실시간으로 산출합니다.
    *   **동기식 체결 확인**: 주문 전송 후 KIS 서버의 응답을 즉시 받아내어, 성공(rt_cd: 0) 시에만 내부 포지션 상태를 업데이트함으로써 데이터 불일치를 원천 차단했습니다.

3.  **🛡️ Fail-Safe 및 리스크 관리**
    *   **자본 보호**: 주문 실패(증거금 부족, 장외 시간 등) 시 즉시 로그를 남기고 다음 틱으로 넘어가며 자본을 보호합니다.
    *   **Bearer 토큰 자동 장착**: 모든 주문은 `KisAuthManager`를 거쳐 가장 최신의 유효한 토큰을 헤더에 장착하고 나갑니다.
