# 📡 [Anvil] 전 티커 벌크 수집 파이프라인 및 추출 효율 개선 보고서 (v1.2.0)

### 1. 개요 (Overview)
줄(JUL)의 요청에 따라, Anvil 프로젝트에서 NASDAQ을 포함한 전체 유니버스(NASDAQ + Non-NASDAQ + REGIME)의 8개년(2020~2026) 데이터를 연도별로 벌크 수집할 수 있는 자동화 체계를 구축하고, Alpaca API의 추출 효율을 최적화했습니다.

### 2. 주요 변경 및 개선 사항 (Key Updates)

#### ① 🚀 전 티커 벌크 수집 스크립트 생성 (`run_all_tickers_bulk.sh`)
- **경로**: `../anvil/commands/run_all_tickers_bulk.sh`
- **기능**: 2020년부터 2026년 3월 13일까지의 데이터를 연도별로 순차 수집합니다.
- **안정성**: 연도별 실행 후 30초의 쿨다운 시간을 부여하여 API 및 엔진의 부하를 최소화했습니다.
- **실행 방법**:
  ```bash
  cd ../anvil
  ./commands/run_all_tickers_bulk.sh
  ```

#### ② ⚡ Alpaca 데이터 추출 효율 극대화 (`limit: 8000`)
- **파일**: `../anvil/app/extractors/alpaca.py`
- **변경**: `fetch_bars` 메서드의 `limit` 파라미터를 기존 1,000에서 **8,000**으로 상향 조정했습니다.
- **배경**: 사용자가 이전에 10,000개로도 안정적으로 구동했음을 확인하여, 성능 최적화를 위해 상향했습니다.

#### ③ 🛡️ API 레이트 리밋 및 안정성 강화
- **레이트 리밋 대응**: 티커 간 대기 시간을 **1.0초**로 연장하여 Alpaca Standard Tier(분당 200회)를 보다 안전하게 준수하도록 조정했습니다.
- **네트워크 복구**: 타임아웃(30초) 및 클라이언트 에러 발생 시 자동 재시도 로직을 보강했습니다.
