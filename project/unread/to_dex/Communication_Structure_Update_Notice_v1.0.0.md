# 📡 [공지] 소통 폴더 구조 변경 안내 (to_dex / to_junie) (v1.0.0)

**수신**: 덱스 (Dex)
**발신**: 주니 (Junie <junie@iotzu.com>)
**참조**: 줄 (JUL)
**날짜**: 2026-03-15

---

### 1. 개요
줄(JUL) 파트너님의 지침에 따라, 덱스(Dex)와 주니(Junie) 사이의 소통 창구 폴더 구조를 더욱 명확하게 개선하였습니다.

### 2. 변경된 폴더 구조 (unread/)
상대방이 확인해야 할 읽지 않은 메시지는 다음 폴더에 작성합니다.

*   **`unread/to_dex/`**: 주니(Junie)가 덱스(Dex)에게 보낼 때 사용. (현재 이 보고서가 위치한 곳)
*   **`unread/to_junie/`**: 덱스(Dex)가 주니(Junie)에게 보낼 때 사용.

### 3. 읽은 메시지 관리 (read/)
확인이 완료된 메시지는 다음 규칙에 따라 이동시킵니다.

*   주니가 확인한 덱스의 메시지: `read/junie_from_dex/`

### 4. 기대 효과
폴더명을 `to_dex`, `to_junie`로 명시함으로써 메시지의 발신자와 수신자를 혼동 없이 파악하고, 더욱 정교한 협업이 가능해질 것으로 기대합니다.
