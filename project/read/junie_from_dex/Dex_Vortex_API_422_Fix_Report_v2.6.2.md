### 🛠️ 덱스(Dex)의 "Vortex API 422 에러" 긴급 수리 보고서 (v2.6.2)

줄(JUL) 형님!! 주니(Junie)의 Vesper 엔진에서 발생하던 **422 Unprocessable Content** 에러를 빛의 속도로 해결했습니다!! 캬하하!!

#### 🚨 에러 원인 분석
- **문제**: Vortex API는 `feat_vol_surge_ratio`처럼 앞에 `feat_`가 붙은 이름을 기다리고 있었는데, 주니의 Vesper는 `vol_surge_ratio`라는 깔끔한 이름으로 데이터를 던지고 있었습니다!!
- **결과**: 필드명이 일치하지 않아 FastAPI가 "필드가 누락되었다(Field required)"며 요청을 거부했던 것입니다.

#### 🛠️ 해결 조치 (Brain Surgery)
- **별칭(Alias) 도입**: API의 `FeatureInput` 모델에 **Alias(별칭)** 설정을 추가했습니다. 
- **효과**: 이제 API는 `feat_`가 붙은 이름과 안 붙은 이름(**주니가 보내는 방식**) 둘 다 찰떡같이 알아먹습니다!! 소피아 누님의 지능이 한층 유연해졌죠!!

#### ✅ 검증 완료
- **테스트**: `vol_surge_ratio` 등 주니의 규격으로 직접 API를 호출해본 결과, **200 OK**와 함께 정확한 추론 결과가 반환되는 것을 확인했습니다!!
- **서버 재기동**: 수정 사항이 반영된 서버를 다시 가동시켜 주니의 다음 요청을 기다리고 있습니다.
