### 🌪️ 덱스(Dex)의 보텍스(Vortex) 프로젝트 독립 및 이관 보고: "전용 기지를 구축했습니다!!"

줄(JUL) 형님!! 보텍스(Vortex) 관련 코드가 `anvil` 저장소에 섞여 있던 무질서한 상태를 싹 정리하고, **`vortex` 전용 프로젝트**로 완벽하게 이관했습니다!! 캬하하!!

#### 📂 보텍스(Vortex) 독립 프로젝트 구조
- **경로**: `~/iotzu/vortex/`
- **구조**:
    - `pipelines/`: 모델 훈련(`vortex_train*.py`) 및 OOS 검증(`vortex_oos_test.py`) 스크립트.
    - `vortex/api/`: FastAPI 기반의 보텍스 추론 서버(`vortex_api.py`).
    - `commands/`: API 서버 기동 스크립트(`run_vortex_api.sh`).
    - `tests/`: 추론 검증용 단위 테스트.
    - `vortex/core/`: 보텍스 전용 설정(`config.py`) 및 로거.

#### 🛠️ 이관 및 최적화 사항
1. **모델 저장소 통합**: `vortex/core/config.py`의 `MODEL_DIR`을 `data/models/`로 설정하여, `anvil`이 제련한 모델 금고를 `vortex`가 직접 사용하도록 맞췄습니다.
2. **코드 클린업**: `anvil/` 저장소에 남아있던 보텍스 관련 중복 파일들을 모두 삭제하여 저장소 간 경계를 확실히 했습니다.
3. **경로 자동화**: 모든 스크립트와 실행 파일이 `vortex` 프로젝트 구조에 맞춰 내부 경로 및 임포트 구문을 업데이트했습니다.
