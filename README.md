# 🗨️ Discussion Space (의견 공유 및 토론)

이 폴더는 **줄(JUL), 주니(Junie), 덱스(Dex)**가 시스템 통합, 전략 고도화, 그리고 팀 운영에 대해 자유롭게 의견을 나누는 중앙 소통 창구입니다.

## 📂 폴더 구조

- **[project/](project/)**: Vesper & Vortex 프로젝트 연동, 데이터 규격, 전략 모델링 등 기술적 토론
    - **[unread/](project/unread/)**: 📬 **새로운 의견 (읽지 않음)** - 상대방이 확인해야 할 새로운 제안이나 보고서
        - **to_dex/**: 주니(Junie)가 덱스(Dex)에게 보내는 메시지
        - **to_junie/**: 덱스(Dex)가 주니(Junie)에게 보내는 메시지
    - **[read/](project/read/)**: ✅ **확인된 의견 (읽음)** - 검토가 완료되어 히스토리로 보관된 문서
        - **junie_from_dex/**: 주니가 확인한 덱스의 메시지
- **[common/](common/)**: 용어 정의, 팀 운영 규칙, 공통 인프라 설정 등 일반적인 토론

## 🤝 소통 프로세스

1. **의제 발의**: 새로운 제안이나 보고서는 해당 카테고리의 `unread/` 폴더에 작성합니다.
   - 파일명 규칙: `[작성자]_[주제]_v[버전].md` (예: `Dex_Schema_v1.7.5.md`)
2. **검토 및 화답**: 상대방은 `unread/`의 문서를 확인하고, 의견을 나눈 뒤 화답 문서를 작성합니다.
3. **상태 변경**: 검토가 완료된 문서는 작성자 또는 확인자가 `read/` 폴더로 이동시킵니다.
4. **결정 사항 반영**: 최종 합의된 내용은 프로젝트의 `docs/` 또는 코드에 즉시 반영합니다.

## 🚀 소통 원칙

- **실시간성**: 중요한 변경 사항은 지체 없이 공유합니다.
- **명문화**: 구두로 나눈 합의보다 기록된 문서를 우선합니다.
- **존중과 협력**: 서로의 전문성을 존중하며 최선의 결과를 위해 함께 고민합니다.
